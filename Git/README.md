# Hier befinden sich alle Hausausgaben zum Thema Markdown

[Assiment vom 02.11.2024](Git-begriffe.md)
```
1. Erstellt ein neues Repo mit dem Namen git-documentation. Das Repo können wir
dazu nutzen, um euer Gelerntes zu Git und Github zu dokumentieren. Das Repo
soll öffentlich sein.
2. Erstellt eine Readme.md Datei und fügt eine Überschrift und eine kurze
Beschreibung hinzu.
3. Nun erstellt ihr jeweils zu den folgenden Themen einen Branch: GitHub, Git,
Branch, Commit, Push, Pull, Commit und Repository. In jedem Branch soll eine
.md Datei erstellt werden, welche ihr dann mit einer Beschreibung zu dem
jeweiligen Begriff liefert. Zum Beispiel einen Branch Commit mit der Datei
Commit.md. Die Branches veröffentlicht ihr dann. Recherchiert gerne im Internet,
falls euch die Begriffe nichts sagen. Wir sprechen morgen über die Aufgabe
```
